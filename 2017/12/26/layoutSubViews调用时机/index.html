layoutSubViews调用时机



这个问题来自于今天在贝贝网面试的一个问题。当时问题是：



在iOS获取网络回调后需要对页面重新布局，页面在什么时机重新布局view？



当时对这个问题一头雾水，我的回答是在网络数据解析完成后根据返回值对页面上的相关的view作重新布局。



可是考官说我并没有回答道点子上。



在今天的面试中，其实有好几个问题我都没有理会到考官所要表述的问题，有点答不对问，在对iOS的理解度上，一些需要刨根问底的细节上，我需要去反思。





回归正题，我在网上搜索layout subviews调用时机，通过对比，选择了一篇文章做为参考，感谢他 给我在这一问题上的一些解答，下面是他的链接。



http://www.cnblogs.com/YouXianMing/p/3897543.html?utm_source=tuicool&utm_medium=referral



下面是这篇文章整理的结论。



/*

在这里我将一一的去做验证…



首先，我将创建1个名为BobTestView的测试view，下面是实现方法



@implementation BobTestView

#pragma mark init
- (instancetype)initWithFrame:(CGRect)frame{
    self = [super initWithFrame:frame];
    if (self) {
        NSLog(@"views initframe    %@",NSStringFromCGRect(frame));
    }
    return self;
}
- (instancetype)init{
    self = [super init];
    if (self) {
        NSLog(@"view init");
    }
    return self;
}
- (void)layoutSubviews{
    [super layoutSubviews];
    NSLog(@"init has been call");
}


1.init初始化不会触发layoutSubviews ？



创建一个根控制器来初始化view，下面是部分方法。



- (void)viewDidLoad {
    [super viewDidLoad];
    [self test];
}
- (void)test{
    // 添加一个测试的view
    BobTestView  view = [[BobTestView alloc]init];
}


以下是输出



2017-12-25 21:15:20.169624+0800 LayOutSubViewsTest[12137:436466] views initframe   {{10, 10}, {10, 10}}


说明init和initwithframe都不会出发layoutSubviews。



2.addSubview会触发layoutSubviews (但是view的frame不能是CGRectZero)?



修改test方法，增加addSubview



- (void)test{
    // 添加一个测试的view
    BobTestView  view = [[BobTestView alloc]initWithFrame:CGRectZero];
    [self.view addSubview:view];
}


以下是输出



2017-12-25 21:19:19.421137+0800 LayOutSubViewsTest[12166:438501] views initframe   {{0, 0}, {0, 0}}
2017-12-25 21:19:26.951894+0800 LayOutSubViewsTest[12166:438501] init has been call


说明addSubview出发了layoutSubviews，而且view的frame是CGRectZero，说明view的frame是CGRectZero也会触发！



3.改变一个UIView的Frame会触发layoutSubviews，当然前提是frame的值设置前后发生了变化 ?



增加一个按钮，用于改变view的大小。



- (void)viewDidLoad {
    [super viewDidLoad];
    [self test];
    [self.view addSubview:self.globeButton];
}
#pragma mark LazyLoad Method
- (UIButton )globeButton{
    UIButton  btn = [[UIButton alloc]initWithFrame:CGRectMake(80, 200, 80, 80)];
    [btn setBackgroundColor:[UIColor blueColor]];
    [btn addTarget:self action:@selector(changeViewSize) forControlEvents:UIControlEventTouchUpInside];
    return btn;
}
#pragma mark Action Method
- (void)changeViewSize{
    self.globeView.frame = CGRectMake(80, 80, 20, 20);
}


以下是输出结果



2017-12-25 21:29:29.775640+0800 LayOutSubViewsTest[12278:445364] views initframe   {{80, 80}, {80, 80}}
2017-12-25 21:29:29.886629+0800 LayOutSubViewsTest[12278:445364] init has been call
2017-12-25 21:29:41.942197+0800 LayOutSubViewsTest[12278:445364] init has been call


可以看到，layoutSubviews两次被触发，说明frame大小发生改变会触发layoutSubviews。

6. 旋转子UIView触发父UIView的layoutSubviews事件 ?



我们在问题5的基础上加一个计时器来修改子视图的frame



- (void)test_3{
    _timer = [NSTimer scheduledTimerWithTimeInterval:1.f
                                              target:self
                                            selector:@selector(timerEvent:)
                                            userInfo:nil
                                             repeats:YES];
    _largeView = [[BobSecondTestView alloc]initWithFrame:self.view.bounds];
    [self.view addSubview:_largeView];
    _smallView = [[BobTestView alloc]initWithFrame:CGRectMake( 80, 80, 80, 80)];
    _smallView.backgroundColor = [UIColor grayColor];
    [_largeView addSubview:_smallView];
}
- (void)timerEvent:(id)sender{
    _smallView.frame = CGRectMake(arc4random()%100 + 20,
                                  arc4random()%100 + 20,
                                  arc4random()%100 + 20,
                                  arc4random()%100 + 20);
}
