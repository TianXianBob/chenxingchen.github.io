cocoaPods私有库——打造专属的公共组建平台


cocoaPods私有库——打造专属的公共组建平台



下面是我通过GitHub来创建私有的pods的具体步骤。



创建两个git仓库



个人练手的话比较推荐GitHub做我们的服务器，当然项目是public状态的。如果你想创建私有仓库，可以通过GitHub，GitLab来创建私有仓库。



两个仓库分别作为私有的Spec Repo和公共组件。



我的sepc repo为名为bobrepo的仓库



我的公共组建仓库为名为bobcomponents的仓库



Spec Repo的本质是一个存储pods的文件夹。“他是所有的Pods的一个索引，就是一个容器，所有公开的Pods都在这个里面，他实际是一个Git仓库remote端。



在GitHub上，当你使用了Cocoapods后他会被clone到本地的/.cocoapods/repos目录下，可以进入到这个目录看到master文件夹就是这个官方的Spec Repo了。”





添加私有仓库到cocoapods



打开终端



添加私有库到cocoapod具体的命令如下



$ pod repo add REPONAME SOURCEURL



REPONAME ：私有仓库的名字（起的简明易懂点在后面还会用到）



SOURCEURL：在GitHub上所创建的



我的REPONAME命名为bobChan，SOURCEURL为https://github.com/TianXianBob/bobrepo.git



$  pod repo add bobChan https://github.com/TianXianBob/bobrepo.git
Cloning spec repo bobChan from https://github.com/TianXianBob/bobrepo.git
$  ls ~/.cocoapods/repos
bobChan           master


可以看到bobChan被成功clone到本地。





克隆组件仓库到本地



在我的demo中我将bobcomponents组件库克隆到本地



命令行克隆



$  git clone https://github.com/TianXianBob/bobcomponents


或者通过sourceTree







制作组件podspec



我们在克隆下来的bobcomponents文件夹中创建一个bobcomponents.podspec



注意：文件名务必和组件项目名一致



下面是一份模板，可按照下面的进行修改



注意的是: spec.source_files 这一项如果不为空，那么对应文件夹不能为空 



Pod::Spec.new do |spec|
#项目名称
spec.name         = 'bobcomponents'
#版本号
spec.version      = '0.1.1'
#开源协议
spec.license      = 'MIT'
#对开源项目的描述
spec.summary      = 'bobcomponents is a request util based on AFNetworking'
#开源项目的首页
spec.homepage     = 'https://github.com/TianXianBob/bobcomponents&#39;
#作者信息
spec.author       = {'bob' => 'bobcxc@163.com'}
#项目的源和版本号
spec.source       = { :git => 'https://github.com/TianXianBob/bobcomponents.git&#39;, :tag => '0.1.1' }
#源文件，这个就是供第三方使用的源文件
spec.source_files = "source/*"
#适用于ios7及以上版本
spec.platform     = :ios, '7.0'
#使用的是ARC
spec.requires_arc = true
#依赖AFNetworking2.0
#spec.dependency 'AFNetworking', '~> 2.0'
end



注意 在podspec文件中所写的版本号需要和github上的分支名一致。





验证并推送到服务器



验证



$  pod lib lint
-> bobcomponents (0.1.1)

bobcomponents passed validation.


注意：验证的时候，会获取BGNetwork.podspec文件中的spec.source来获取git服务器上面对应版本的代码，然后再找到spec.sourcefiles中的源代码，通过xcode命令行工具建立工程并且进行编译。所以这一步的过程会比较久，如果编译没有错误，就验证通过。



推送



$  pod repo push REPO_NAME SPEC_NAME.podspec
Validating spec
 -> bobcomponents (0.1.1)

Updating the bobChan&#39; repo

Your configuration specifies to merge with the ref &#39;refs/heads/master&#39;
from the remote, but no such ref was fetched.

Adding the spec to thebobChan' repo

 - [Add] bobcomponents (0.1.1)

Pushing the `bobChan' repo


REPO_NAME:我这里是bobChan
SPEC_NAME.podspec:我这里是bobcomponents.podspec

如果没有错误，但是有警告，我们就将警告解决，也可以加—allow-warnings来提交
